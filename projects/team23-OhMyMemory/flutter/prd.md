# Frontend (Flutter) PRD: 추억 속 음악 추천 에이전트

## 1. 프로젝트 개요
- **서비스 목적**: 사용자의 취향과 세대/지역 맥락을 반영하여 잊고 있던 '추억의 향수곡'을 찾아주는 에이전트 서비스.
- **플랫폼 타겟**: Web, 토스 미니앱, Android(.apk)를 지원하는 크로스 플랫폼 단일 뷰(Single View) 앱.
- **프론트엔드 핵심 역할**: 직관적인 카드 스와이프 UI 구현, 오디오 재생 제어, 사용자 반응(Feedback) 데이터 수집 및 API 통신.

## 2. 핵심 구현 기능 (MVP 기준)

### A. 온보딩 및 로그인 화면
- **인증 UI**: 소셜 로그인 및 회원가입 버튼. (비회원도 사용 가능하도록 '건너뛰기' 기능 포함)
- **대화형 온보딩 UI**: 나이대, 선호 장르, 선호 가수 등을 채팅 혹은 단계별 폼 형태로 입력받는 화면.

### B. 메인 추천 화면 (단일 뷰 중심)
- **카드 스와이프 UI**: 백엔드에서 전달받은 5~7개의 추천 곡 리스트를 틴더(Tinder) 형식의 카드 스와이프나 슬라이더로 넘겨볼 수 있게 구현.
- **음악 메타데이터 표시**: 앨범 아트 이미지, 곡명, 아티스트, 그리고 추천 맥락인 '감성 타이틀' (예: "2016년 겨울밤 감성")을 카드 내에 렌더링.
- **미리듣기 오디오 플레이어**: 백엔드에서 제공하는 30초 음원 URL을 재생/일시정지할 수 있는 오디오 컨트롤러 기능.
- **아웃링크 버튼**: 클릭 시 유튜브 뮤직 등 외부 음원 스트리밍 앱으로 연결되는 딥링크(Deep Link) 구현.

### C. 사용자 반응(Reaction) 수집 UI
- **상태 선택 버튼**: 곡 하단에 [알아요], [듣고 싶어요], [몰라요] 3가지 핵심 반응 버튼 구현.
- **추가 평가 UI**: 좋아요/싫어요 토글 버튼 및 별점(1~5점) 입력 컴포넌트.
- **조건부 피드백(채팅/입력) 창**: 사용자가 [몰라요]를 **연속 3회** 누를 경우를 감지하여, 꼬리 질문을 띄워주거나 자연어 피드백("더 잔잔한 곡으로 해줘" 등)을 입력받을 수 있는 입력창(Bottom Sheet 또는 Dialog) 팝업 처리.

### D. 보관함 및 상태 관리
- **마이페이지 / 보관함 뷰**: 마음에 든다고 표시한 곡들을 모아보는 리스트 화면.
- **로컬 스토리지 (Local Storage)**: 비회원 상태일 때 세션 데이터(반응 신호, 임시 보관함 등)를 유지하기 위한 로컬 데이터 영속화(SharedPreferences, Hive 등 활용). 세션 종료 시 초기화되도록 로직 구성.
