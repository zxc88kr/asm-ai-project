# 피드백 기능 품질 개선 지침 (Advanced)

> 이 문서는 `feedback-feature-prompt.md`에서 정의한 피드백 루프의 성능을 극대화하고, LLM이 복합적인 수정 요청을 정확히 처리하도록 보완하는 기술 지침이다.

## 1. 정밀한 피드백 파싱 (Direct Override 지원)

`feature-prompt`의 `_FEEDBACK_PARSING_SYSTEM`이 처리하지 못하는 "최종 금액 직접 지정" 및 "상대적 비율 수정" 로직을 강화한다.

*   **최종 금액 강제 지정 (`fixed_amount`)**:
    *   사용자가 "A는 2만원만 내게 해줘"라고 할 경우, 이를 단순 `discount_rate`가 아닌 고정값으로 파싱하도록 JSON 구조를 확장한다.
    *   `parsed_json` 내 `participants` 요소에 `fixed_amount` 필드(Optional)를 추가 유도.
*   **복합 조건 처리**: 
    *   "A가 안주를 덜 먹었고(0.5), 술은 아예 안 마셨어(0)"와 같은 다중 조건이 한 번에 입력될 때, 기존 `parsed_json`의 해당 필드를 누적/교체하는 명확한 우선순위를 프롬프트에 명시한다.

## 2. 피드백 이력(History) 관리 및 일관성 유지

피드백이 여러 번 반복될 때 발생할 수 있는 '지시사항 충돌'을 방지한다.

*   **컨텍스트 요약**: `feedback_history`가 길어질 경우, LLM이 최신 지시사항에 우선순위를 두도록 프롬프트를 구성한다. 
    *   예: "이전 피드백과 현재 피드백이 충돌할 경우, 가장 최근의 지시를 따른다."
*   **데이터 정합성 보정**: `feedback_parsing_node` 이후, 수정된 `parsed_json`의 총합이나 논리적 모순(예: 참여자 수보다 지각자가 많은 경우 등)을 체크하는 `logic_validator` 로직을 `safety_check` 단계에 결합한다.

## 3. UI/UX: 비교 및 추적 기능 (Gap 2-3 보완)

단순 결과 출력을 넘어, 사용자가 무엇이 바뀌었는지 인지하게 한다.

*   **변경 사항 하이라이트**: `report_generation_node`에서 `feedback_intent == "modify_exception"`일 경우, 이전 결과와 달라진 부분(주로 금액이 변동된 인물)에 텍스트 강조(예: `**23,000원** (← 28,000원)`)를 적용한다.
*   **수정 근거 명시**: LLM이 왜 이렇게 수정했는지 요약 한 줄을 `calc_explanation`에 포함시킨다.
    *   예: "C님의 지각비 제외 요청을 반영하여 재계산했습니다."

## 4. `complaint` 의도의 지능적 대응

"계산이 이상해"라는 모호한 입력에 대해 단순히 되묻는 것을 넘어, 예상되는 오류 지점을 먼저 제안한다.

*   **자가 진단(Self-Diagnosis)**: `feedback_intent_node`에서 `intent`가 `complaint`인 경우, LLM에게 "현재 계산 결과에서 사용자가 의문을 가질만한 복잡한 규칙(예: 여러 차례 적용된 할인 등)이 무엇인지"를 1순위로 찾아내게 하여 되묻기 메시지에 포함시킨다.
    *   예: "혹시 D님의 주류 미섭취 할인(0.5)이 너무 많이 적용되었다고 생각하시나요? 구체적인 이유를 말씀해주세요."

## 5. 구현 우선순위 및 예외 처리

1.  **우선순위**: `fixed_amount` 파싱 지원 > 피드백 이력 일관성 > 변경 사항 하이라이트.
2.  **예외**: 사용자가 피드백 모드에서 완전히 새로운 정산을 입력했는데 `intent`가 `modify_exception`으로 오분류된 경우, 파싱된 참여자가 기존과 50% 이상 다르면 자동으로 `reset`으로 간주하는 가드 로직을 `ai/nodes.py`에 추가한다.
