동네 기반 소모임·예약 플랫폼을 만드는 초기 스타트업에서 프론트엔드 인턴으로 근무했습니다.

### Key Contributions

- Next.js, TypeScript 기반 MVP 프론트엔드를 7주 안에 구현하고 베타 출시
- 회원가입, 카카오 로그인, 모임 탐색, 예약 신청, 예약 관리, 호스트 대시보드 화면 개발
- 사용자가 모임을 탐색하고 예약하기까지의 funnel event를 설계하고 GA와 Mixpanel로 추적
- 예약 신청 과정에서 이탈률이 높은 입력 단계를 줄이고 CTA copy를 개선하여 예약 전환율 상승
- 지역별 모임 목록과 인기 모임 section을 구현하고 skeleton loading, empty state, error state 정리
- React Query cache와 prefetch를 적용하여 모임 상세 페이지 진입 체감 속도 개선
- 호스트가 예약자를 관리할 수 있는 dashboard와 승인·거절 UI 구현
- 사용자 인터뷰 16건을 바탕으로 "모임 위치와 시간 정보가 부족하다"는 문제를 발견하고 상세 페이지 정보 구조 개선
- 모바일 사용자가 80% 이상인 점을 고려해 bottom sheet, sticky CTA, one-hand interaction 중심으로 UI 개선
- 디자이너 없이 Figma wireframe을 직접 제작하고 프론트엔드 구현까지 주도

### Startup Impact

- MVP 출시 후 6주간 가입자 380명 확보
- 모임 상세 진입 후 예약 신청 전환율 18%에서 32%로 개선
- 온보딩 완료율 61%에서 79%로 개선
- 모임 상세 페이지 평균 로딩 시간 2.3초에서 1.0초로 개선
- 사용자 인터뷰 기반 UI 개선 14건 반영
- 2주 단위 product iteration 5회 참여

### Tech Stack

Next.js, React, TypeScript, React Query, Zustand, Tailwind CSS, Framer Motion, Vercel, Google Analytics, Mixpanel

### Project 1. LocalMeet — 지역 기반 모임·예약 플랫폼

GitHub: https://github.com/minjae-startup-frontend/localmeet  
Demo: https://localmeet.dev  
Period: 2025.01 - 2025.04  
Role: Founder / FrontEnd Engineer

지역 기반으로 소모임을 만들고, 사용자가 날짜별로 신청·예약할 수 있는 커뮤니티 플랫폼입니다.

### Problem

지역 모임 서비스에서는 사용자가 모임 정보를 빠르게 이해하고, 신뢰할 수 있는 정보를 바탕으로 예약을 완료하는 것이 중요합니다. 하지만 초기 서비스에서는 탐색, 상세 확인, 예약 신청 과정이 길어지면 사용자가 쉽게 이탈합니다.

### Solution

- 모바일 우선 모임 탐색 UI 구현
- 지역, 카테고리, 날짜 기반 필터 제공
- 모임 상세 페이지에서 장소, 시간, 호스트 정보, 남은 자리 수를 명확히 표시
- 예약 CTA를 sticky button으로 제공
- 예약 신청 과정을 3단계에서 1단계로 줄임
- 호스트 대시보드와 관리자 신고 처리 화면 구현

### Key Contributions

- 전체 프론트엔드 구조 설계 및 페이지 24개 구현
- Next.js 기반 routing과 SEO metadata 구성
- React Query로 모임 목록, 상세, 예약 상태 관리
- Zustand로 필터, bottom sheet, modal 등 UI 상태 관리
- 예약 신청 optimistic update와 실패 시 rollback 처리 구현
- 사용자 행동 event 설계: view_meeting, click_reserve, submit_reserve, cancel_reserve
- GA와 Mixpanel을 연동하여 funnel drop-off 분석
- 모바일 UX 개선을 위해 bottom navigation과 sticky CTA 구현
- Figma wireframe 제작 후 사용자 테스트 12회 진행
- Playwright로 예약 신청, 취소, 호스트 승인 flow 테스트 작성

### Impact

- 베타 사용자 210명 확보
- 모임 생성 84건, 예약 신청 430건 발생
- 예약 신청 전환율 19%에서 34%로 개선
- 온보딩 완료율 57%에서 78%로 개선
- 모임 상세 페이지 LCP 3.4초에서 1.5초로 개선
- 모바일 Lighthouse Performance 64점에서 90점으로 개선
- 사용자 피드백 기반 product iteration 9회 수행

### Tech Stack

Next.js, React, TypeScript, React Query, Zustand, Tailwind CSS, Framer Motion, Playwright, Vercel, Google Analytics, Mixpanel

### Project 2. LaunchCart — 소규모 브랜드용 커머스 프론트엔드

GitHub: https://github.com/minjae-startup-frontend/launchcart  
Demo: https://launchcart.dev  
Period: 2024.10 - 2024.12  
Role: FrontEnd Developer

소규모 브랜드가 상품을 판매하고 주문을 관리할 수 있는 경량 이커머스 웹 서비스입니다.

### Problem

초기 브랜드는 복잡한 커머스 솔루션보다 빠르게 상품을 등록하고 판매할 수 있는 단순한 웹 서비스가 필요합니다. 사용자는 모바일에서 상품을 빠르게 확인하고 주문까지 완료할 수 있어야 합니다.

### Solution

- 상품 목록, 상품 상세, 장바구니, 주문서, 주문 완료 화면 구현
- 브랜드 운영자를 위한 상품 등록, 주문 관리 화면 구현
- 결제 API 연동을 가정한 주문 상태 UI 구성
- 모바일 구매 flow를 단순화하여 주문 완료율 개선
- 이미지 lazy loading과 skeleton UI로 체감 속도 개선

### Key Contributions

- React, TypeScript 기반 커머스 화면 구현
- React Hook Form과 Zod를 활용해 주문서 validation 구현
- 장바구니 상태를 local storage와 Zustand로 관리
- 주문 완료 후 상태 초기화와 중복 제출 방지 처리
- 상품 옵션 선택 UX 개선
- 관리자 주문 목록에 상태 필터와 검색 기능 구현
- 상품 이미지 최적화와 lazy loading 적용
- API mock을 MSW로 구성하여 백엔드 API 완료 전 개발 진행
- Swagger 기반 API 명세를 바탕으로 타입 정의

### Impact

- 테스트 사용자 60명 기준 주문서 완료율 48%에서 67%로 개선
- 상품 상세 페이지 평균 로딩 시간 2.1초에서 1.0초로 개선
- 주문서 validation 오류 관련 사용자 문의 감소
- 관리자 주문 처리 화면 개발로 주문 확인 시간 단축
- 모바일 구매 flow 사용성 만족도 4.4 / 5.0 기록

### Tech Stack

React, TypeScript, Vite, Zustand, React Hook Form, Zod, Tailwind CSS, MSW, Vercel

### Project 3. TeamPulse — 팀 업무 공유 및 알림 웹 서비스

GitHub: https://github.com/minjae-startup-frontend/teampulse  
Period: 2024.07 - 2024.09  
Role: Full-stack / FrontEnd Focus

작은 팀이 업무를 공유하고 댓글, 멘션, 마감일 알림을 받을 수 있는 협업 도구입니다.

### Problem

초기 팀에서는 업무 공유가 메신저 중심으로 이루어져 담당자, 마감일, 변경 이력이 누락되기 쉽습니다. 업무 상태를 한눈에 보고 빠르게 수정할 수 있는 UI가 필요했습니다.

### Solution

- 프로젝트, 태스크, 담당자, 마감일, 댓글을 한 화면에서 관리
- 드래그 앤 드롭 기반 task board 구현
- 멘션과 댓글 알림 UI 구현
- 마감일 임박 task를 강조 표시
- 팀 단위 workspace와 권한별 화면 접근 분리

### Key Contributions

- React 기반 task board UI 구현
- dnd-kit을 활용해 드래그 앤 드롭 task status 변경 구현
- WebSocket 기반 실시간 알림 prototype 구현
- 댓글 작성 시 멘션 autocomplete UI 구현
- task detail drawer와 quick edit UI 구현
- 프로젝트별 통계 chart 구현
- optimistic update로 task 이동 체감 속도 개선
- 사용성 테스트를 통해 task 생성 form 필드를 7개에서 4개로 축소

### Impact

- 팀 사용자 7개 그룹, 총 64명 테스트
- 태스크 생성 후 담당자 지정률 37%에서 68%로 개선
- task status 변경 평균 소요 시간 8초에서 3초로 단축
- 마감일 누락 피드백 40% 감소
- 실시간 알림 적용 후 댓글 확인 지연 감소

### Tech Stack

React, TypeScript, dnd-kit, Zustand, React Query, WebSocket, Tailwind CSS, Recharts, Vercel
