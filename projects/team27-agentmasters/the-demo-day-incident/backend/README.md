# The Demo Day Incident Backend

추리 게임 `The Demo Day Incident`의 백엔드 레포입니다. 게임 진행에 필요한 API, 데이터 모델, 서버 로직을 담당합니다.

## 역할

- 게임 진행 API 구현
- 사건, 단서, 증언, 인물 데이터 모델 관리
- 추리 결과 판정과 진행 상태 처리
- 인증, 저장소, 배포 환경 등 서버 영역 관리
- 프론트엔드와 합의한 API 계약 구현

공통 설계, 게임 규칙, API 계약 초안은 부모 레포인 `the-demo-day-incident`에서 관리합니다.

## 작업 규칙

### Issue 먼저 생성

모든 작업은 GitHub Issue를 먼저 생성한 뒤 시작합니다. 작업 브랜치는 Issue 번호를 기준으로 만듭니다.

```text
<prefix>/<issue-num>
```

예시:

```text
feat/12
fix/24
docs/31
```

### PR 대상

백엔드 구현, API 수정, 데이터 모델 변경, 서버 테스트와 관련된 작업은 이 `backend` 레포에 PR을 생성합니다.

설계 변경, API 계약 변경, 게임 규칙 변경처럼 프론트엔드 또는 전체 프로젝트에 영향을 주는 내용은 부모 레포에 먼저 문서화합니다.

## 커밋 컨벤션

커밋 메시지는 Conventional Commits 형식을 사용하고, 설명은 한국어로 작성합니다.

```text
<type>: <한국어 요약>
```

예시:

```text
feat: 단서 조회 API 추가
fix: 추리 결과 판정 조건 오류 수정
docs: 백엔드 실행 방법 추가
refactor: 게임 진행 서비스 분리
test: 사건 상태 변경 테스트 추가
chore: 서버 환경 설정 추가
```

## PR 컨벤션

PR 제목은 아래 형식을 따릅니다.

```text
<type>: <한국어 요약>
```

PR 본문에는 아래 항목을 포함합니다.

```markdown
## 작업 내용

- 
```

## 문서 위치

- 백엔드 구현과 실행 방법: 이 레포의 README 또는 관련 문서
- 공통 설계와 협업 규칙: 부모 레포 `the-demo-day-incident`
- 프론트엔드 구현과 클라이언트 실행 방법: `frontend` 레포

## MVP API 참고

- 현재 사용자 구분은 인증 토큰이 아니라 요청 헤더 기반 MVP 구현입니다.
- 권장 헤더는 `X-User-Id`이며, 기존 호환을 위해 `user-id`, `user_id`도 허용합니다.
- 기존 SQLite DB를 사용하는 경우 앱 시작 시 `user_id` 컬럼을 보강하는 최소 마이그레이션을 수행합니다. 운영용 정식 마이그레이션 도구는 아직 도입하지 않았습니다.
