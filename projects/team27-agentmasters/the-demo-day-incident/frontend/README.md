# The Demo Day Incident Frontend

추리 게임 `The Demo Day Incident`의 프론트엔드 레포입니다. 플레이어가 사건을 조사하고, 단서를 확인하며, 추론을 진행하는 사용자 경험을 담당합니다.

## 역할

- 게임 화면과 사용자 인터랙션 구현
- 단서, 증언, 인물, 사건 진행 상태 표현
- 백엔드 API 연동
- 프론트엔드 상태 관리
- 접근성과 반응형 UI 관리

공통 설계, 게임 규칙, API 계약 초안은 부모 레포인 `the-demo-day-incident`에서 관리합니다.

## 작업 규칙

### Issue 먼저 생성

모든 작업은 GitHub Issue를 먼저 생성한 뒤 시작합니다. 작업 브랜치는 Issue 번호를 기준으로 만듭니다.

```text
<prefix>/<issue-num>
```

예시:

```text
feat/12
fix/24
docs/31
```

### PR 대상

프론트엔드 구현, UI 수정, 클라이언트 상태 관리, 프론트엔드 테스트와 관련된 작업은 이 `frontend` 레포에 PR을 생성합니다.

설계 변경, API 계약 변경, 게임 규칙 변경처럼 백엔드 또는 전체 프로젝트에 영향을 주는 내용은 부모 레포에 먼저 문서화합니다.

## 커밋 컨벤션

커밋 메시지는 Conventional Commits 형식을 사용하고, 설명은 한국어로 작성합니다.

```text
<type>: <한국어 요약>
```

예시:

```text
feat: 단서 상세 화면 추가
fix: 인물 카드 선택 상태 오류 수정
docs: 프론트엔드 실행 방법 추가
refactor: 조사 화면 컴포넌트 분리
test: 단서 필터링 테스트 추가
chore: 프론트엔드 빌드 설정 추가
```

## PR 컨벤션

PR 제목은 아래 형식을 따릅니다.

```text
<type>: <한국어 요약>
```

PR 본문에는 아래 항목을 포함합니다.

```markdown
## 작업 내용
- 

## 관련 이슈
- Closes #

## 확인 방법
- 

## 참고 사항
- 
```
